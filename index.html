I stopped collecting AI tools. I started building with them.

I downloaded Gemini CLI, subscribed to Copilot, installed Claude Code, tested Kimi K2, tried GLM 4.6—and used exactly none of them well. This is the rebuild: honest logs of failed prompts, tool comparisons that actually matter, and proof I'm finally shipping instead of scrolling.

Why I'm rebuilding everything

I got tired of performing productivity. Every engineer I admired was shipping AI-augmented features while I was stuck in tutorial hell. Documenting my specific failures—from prompt waste to tool paralysis—became the only way out.

Tool Hoarding

The choice paralysis that broke me

  • •I collected 6 AI agents like Pokemon cards: Gemini CLI, Copilot, Claude Code, Kimi K2, GLM 4.6. Used zero of them in my actual workflow.
  • •Every tutorial promised "10x productivity" but I was too scared to delete my manual processes.
  • •I measured it: 47 hours spent 'evaluating' tools, 0 hours shipping with them. The shame was motivational.

The Hype Spiral

Drowning in potential, starving for proof

  • •I could quote every AI paper but couldn't show one feature I shipped using AI in my day job.
  • •Each new model drop (Gemini Pro 3.0, Kimi K2) made me feel further behind, not more capable.
  • •The breaking point: I caught myself doom-scrolling AI Twitter instead of writing the prompt that would actually fix my broken deployment script.

Workflow Fossilization

The comfort zone that became a cage

  • •My brain still defaults to Google + Stack Overflow, even with Claude Code one terminal tab away.
  • •Every time I typed the same bash command for the 100th time, I heard a voice: "You should automate this." I ignored it 99 times.
  • •The real blocker was emotional: What if the AI-generated code breaks production? My manual scripts felt safe—even when they were slow and wrong.

Roadmap

How I’m rewriting my operating system

Structure keeps me honest. Each phase has specific deliverables, so I can measure whether I’m actually relearning—or just consuming content.

Every log references these checkpoints, so you can trace experiment → lesson → habit.

Phase 0

Kill the Tutorial Habit

Stop consuming, start building. Delete the bookmark folder of "AI tools to try" and pick exactly two that map to my actual day-job pain.

  • •Count hours spent on tools vs. tutorials (target: 80% building, 20% learning)
  • •Document the emotional resistance when reaching for Google instead of AI
  • •Publish the first embarrassing GitHub repo: AI-assisted code with all my mistakes visible
Phase 1

Prove Augmentation Actually Works

Turn AI from a toy into a teammate. Measure real velocity gains on actual cloud deployment tasks, not toy problems.

  • •Time every deployment: manual vs. AI-assisted (minimum 10 tasks each)
  • •Publish specific prompt templates that failed vs. ones that shipped code
  • •Share the bash script that AI wrote and I was too proud to commit—but finally did
Phase 2

Teach the Failure, Not the Success

The market is flooded with "AI changed my life" content. I will document the specific friction points so others can skip my wasted hours.

  • •Open-source my prompt graveyard: 50+ failed attempts with error analysis
  • •Host a live stream: "I'm a cloud engineer who pretended to know AI for 6 months—here is what I actually learned"
  • •Build a comparison matrix: when to use each agent (Gemini vs Claude vs Copilot) based on real tasks, not marketing

Cadence

The experiments that keep me honest

These are not fluffy ideas—they’re recurring rituals on my calendar. They create proof that I’m improving, or evidence that something needs to change.

Grading Myself in Public

Every Sunday I post three scores: hours actually building with AI (target: 15+), prompts shipped vs. prompts hoarded, and moments I reached for Google instead of my AI stack. Week 0 was humiliating (scores: 3, 0, 23).

Agent Duel Logs

I take the same cloud task and run it through two different AI agents. Publish the diff, timing, and my emotional preference (which often contradicts the performance data). Results: Gemini CLI wins on cloud tasks, but I keep reaching for Claude Code.

Prompt Graveyard

For every successful prompt, I log 5-10 failed versions with specific error analysis. The goal is not to look smart—it is to prove I am actually iterating instead of copy-pasting from Twitter.

The Three Pillars

Relearn Life: The OS Crash

Limiting belief: 'If I just push harder, I'll catch up.' Reality: I was doom-scrolling AI Twitter at 2am, sleeping 4 hours, and wondering why prompts felt overwhelming. These journaling and energy systems stopped the spiral.

Relearn Cloud: Fossilized Workflows

The inefficiency: My deployment runbook had 47 manual steps from a 2019 wiki. AI could do it in 4. The emotional block: fear that AI-generated infrastructure code would break production. Proof: I shipped it anyway and it saved 12 minutes.

Relearn AI: My First Real Augmentation

Without AI: 8 hours writing automation scripts, 3 errors in review. With Claude Code: 2.5 hours, zero errors. Exact prompt templates and the GitHub diff are in Week 01 log. This is proof—not hype.
